How To Learn Website Development In 3 Days

(HTML, CSS, JavaScript, Bootstrap) &

Create Your Own Website

Kevin Lin | 5th January, 2019

You'd like to learn Web Development, but where do you start? Youtube videos? Online courses? MOOCS? Paid courses or free courses? A common obstacle to learning web dev is not knowing where to start. I felt that way too. Here's how I navigated these possibilities and learnt the skills to create this website in 3 days.

Here’s how I learnt HTML/CSS/JS/Bootstrap in under 3 days, from almost* no web dev experience.



Day 1: Go to Udacity’s “Intro to HTML and CSS”

@ https://www.udacity.com/course/intro-to-html-and-css--ud001

Complete this free course in less than 1 day

This short course gives you a strong conceptual understanding of HTML and CSS. I completed all the coding exercises. If you find these videos slow (as I did), play them on 2x speed.



Day 2: Go to Udemy’s “The Complete Web Developer Course 2.0”

@ https://www.udemy.com/the-complete-web-developer-course-2/

Complete this ($11**) course in less than 2 days

This course is extremely comprehensive and gives you all the resources to host a website on the internet.

After completing Udacity’s free course, many of the HTML/CSS videos here will feel repetitive. To minimize boredom, play the videos on 2x speed and use the ‘right’ arrow button on your keyboard to fast forward.



Day 3: Download the free bootstrap theme (as part of the $11 you pay) and pick one to tweak

Before tweaking: Plan out how you want to organize your website. This blog post has relevant tips: neilpatel.com/blog/site-structure-enhance-seo/

Once you’ve planned out your website’s structure and tweaked index.html (within “v4/docs”), upload your files and folders using the FTP that the instructor on Udemy describes.



Now, you might run into a little issue during this uploading process. That is, what should you upload?

The main index.html file is stored within several folders. And, by default, your website will display the “index.html” directly inside the “public_html” folder. Here’s what I did: upload only the files and folders within the ‘docs’ folder into “public_html”.



Three days was enough for me to learn how to do tasks specific to making my own website. For example, after the courses, I had to learn to create drop down headers, create sticky footers, create internal links. In this case, “learning” consisted of Google searches, reading, pasting and tweaking code in my text editor. (No worries if you don’t know what a text editor is yet, you’ll find out in the Udemy course.)

If you decide to learn web dev and create your own website, I suspect you’ll also have to keep “learning” in the fashion I’ve described.

Good luck and enjoy the process!



P.s. If you write articles or guides on your website, here’s a great tool that converts word docs to html: https://www.textfixer.com/html/convert-word-to-html.php

* strictly speaking, I did have some ‘experience’, I watched a friend make this un-styled website: https://kevin-thankyou-lin.github.io/

** I’d consider the $11 a great price because, in addition to the comprehensive course from an instructor who graduated from Cambridge, you get free webhosting and a domain name. These combined would cost much more than $11.
